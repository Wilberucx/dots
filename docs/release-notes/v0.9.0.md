# Release Note: v0.9.0 — Python to Go: Complete Rewrite

**Fecha:** 2026-05-21
**Versión:** `v0.9.0`
**Tipo:** Rewrite / Architecture / Performance

## 🎯 ¿Por qué la migración a Go?

`dots` nació en Python porque era rápido de prototipar. Pero a medida que el proyecto creció — más comandos, más dependencias, más usuarios — las limitaciones de Python se hicieron evidentes.

### Problemas con Python/pipx

| Problema | Impacto |
|----------|---------|
| Python 3.10+ obligatorio | Usuarios sin Python instalado no podían usar dots |
| pipx + .venv | Instalación lenta (~30s), requiere pipx + git |
| Dependencias runtime | 10+ dependencias pip, resolución frágil |
| `shutil.which` lento | Cada comando escaneaba PATH |
| Sin binario estático | No se podía distribuir como single binary |
| Actualización manual | `pipx upgrade dots`, necesita git pull |

### Solución: Go

Todo `dots` fue reescrito en Go 1.26 como un solo binario estático. Sin dependencias runtime. Sin Python. Sin .venv.

| Beneficio | Antes (Python) | Ahora (Go) |
|-----------|---------------|------------|
| Instalación | `pipx install` (~30s) | `curl | bash` (~3s) |
| Dependencias | Python 3.10+, pipx, pip, git, 10+ libs | **Cero** — binario estático |
| Actualización | `pipx upgrade dots` | `curl ... install.sh | bash` |
| Tamaño | ~5MB (código + .venv) | ~7MB (único binario) |
| Startup | ~300ms (Python import) | ~5ms (Go compiled) |
| Distribución | pip/git | GitHub Releases (4 platforms) |

## 🛠 Cambios Técnicos

### 1. Arquitectura Go

- **Módulo único** (`cmd/dots/`): Entry point minimalista
- **Paquetes internos** (`internal/cli/`, `internal/yaml/`, `internal/resolver/`, etc.): Lógica pura sin dependencias externas pesadas
- **Cobra** para CLI: El estándar de facto en Go (mismo engine que kubectl, hugo, etc.)

## ⚠️ Breaking Changes

| Aspecto | Antes | Ahora |
|---------|-------|-------|
| Instalación | `pipx install github.com/Wilberucx/dots` | `curl -fsSL https://raw.githubusercontent.com/Wilberucx/dots/main/install.sh \| bash` |
| Actualización | `pipx upgrade dots` | Re-ejecutar install.sh |
| PATH del binario | `~/.local/bin/dots` (pipx) | `~/.local/bin/dots` (mismo) |
| `dots.toml` | Requerido | Requerido (sin cambios) |
| path.yaml v3 | Soportado | Soportado (mismo schema) |

No hay cambios en el formato de `path.yaml` ni en la estructura del repositorio de dotfiles.
