# Release Note: v0.7.5 — Refactor Adopt & Module Services

**Fecha:** 2026-05-06
**Versión:** `v0.7.5`
**Tipo:** Refactor / Bugfix / Architecture

## 🎯 Objetivo
Sanear la arquitectura del comando `adopt`, centralizar la manipulación de metadatos YAML y corregir bugs críticos de detección de rutas en el Schema v3.

## 🛠 Cambios Técnicos

### 1. Nueva Capa de Servicios: `module_writer.py`
Se extrajo toda la lógica de manipulación de archivos `path.yaml` del comando `adopt.py` hacia un nuevo módulo dedicado en `core`.
- **Responsabilidad:** Leer, validar y escribir entradas de archivos en los módulos.
- **Beneficio:** Otros comandos (como `migrate` o `edit`) ahora pueden reutilizar esta lógica sin duplicar código.

### 2. Soporte Nativo para Schema v3 (`per-os`)
Se corrigió un bug donde el comando `adopt` ignoraba las rutas declaradas bajo la llave `per-os`.
- **Fix:** `is_destination_declared` ahora inspecciona recursivamente los valores dentro de `per-os`.
- **Depuración:** Se eliminó explícitamente el soporte para campos v2 obsoletos (`destination-linux`, `destination-mac`) para evitar falsos positivos y mantener la limpieza del esquema actual.

### 3. Deuda Semántica en Transacciones
Se identificó que `adopt` usaba `TransactionLog.backup()` para importar archivos, lo cual era conceptualmente erróneo.
- **Mejora:** Se añadió el método `move(src, dest)` a `TransactionLog`.
- **Rollback Seguro:** El sistema ahora registra explícitamente un "move" y, en caso de fallo, realiza el movimiento inverso para restaurar el archivo a su ubicación original en el HOME del usuario.

### 4. Orquestación Limpia en `adopt_cmd`
El comando en `src/dots/commands/adopt.py` se redujo drásticamente:
- Se eliminaron imports muertos (`yaml`, `shutil`, `os`).
- Se consolidaron los casos de "adopción normal" y "variantes" mediante la función interna `_do_adopt`.
- El comando ahora solo se encarga del flujo de usuario (prompts) y delega la ejecución al sistema de servicios.

## 🧪 Validación
- **Tests Unitarios:** Se añadieron 5 casos nuevos en `tests/test_module_writer.py`.
- **Protección contra Regresiones:** Test específico que garantiza que campos v2 no interfieran con la lógica v3.
- **Suite Completa:** 127 tests ejecutados y aprobados (125 passed, 2 skipped).
