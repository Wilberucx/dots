# Release Note: v0.10.0 — Checker improvements, dangling symlink detection, UI enhancements

**Fecha:** 2026-05-24
**Versión:** `v0.10.0`
**Tipo:** Features / Bug fixes

## 🚀 Nuevas Funcionalidades

### 1. Checker en `dots status`

El syntax checker de `path.yaml` ahora también se ejecuta automáticamente al correr `dots status`, mostrando issues de validación junto con el estado de los módulos. También incluye detección de symlinks rotos (dangling) y destinos inseguros.

### 2. Detección de dangling symlinks

`dots status` ahora detecta symlinks que apuntan a destinos inexistentes (dangling) y los reporta como `broken link` en el checker, en vez de mostrar información confusa.

### 3. UI con estilos consistentes

Los mensajes de error y warning en el checker ahora usan la misma paleta de colores y estilos que el resto de la TUI (vía `internal/ui/`), mejorando la consistencia visual.

## 🐛 Bugs Corregidos

### 1. Symlinks almacenados con `~` literal

`resolveSingleState()` no expandía `~` en el target de symlinks almacenados por el sistema operativo, causando que `filepath.IsAbs()` devolviera `false` y generara paths incorrectos al unirlos con el directorio padre del destino.

**Fix:** Se expande `~` en el target leído del symlink ANTES de evaluar si es absoluto.

### 2. `IsSafePath` no expandía `~`

`system.IsSafePath()` usaba `filepath.Abs()` directamente sobre paths con `~`, lo que resolvía contra el CWD en vez del HOME, marcando incorrectamente paths seguros como inseguros.

**Fix:** Se expande `~` con `ExpandPath()` antes de `filepath.Abs()`.

### 3. Falsa seguridad en `/tmp` y directorios del sistema

Paths en `/tmp` y otros directorios del sistema ahora se detectan correctamente como inseguros gracias al fix de `IsSafePath`.
