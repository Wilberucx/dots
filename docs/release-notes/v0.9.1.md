# Release Note: v0.9.1 — Bug fixes: symlink detection, path expansion, variant naming

**Fecha:** 2026-05-21
**Versión:** `v0.9.1`
**Tipo:** Bug fixes

## 🐛 Bugs Corregidos

### 1. Trailing slash en destinations causaba falsos "unlinked"

Las rutas destino con trailing slash (`~/.config/nvim/`) provocaban que `os.Lstat` siguiera el symlink en Linux en vez de leerlo, reportando `StatePending` en lugar de `StateLinked`.

**Fix:** `strings.TrimRight` del trailing slash antes de cada `os.Lstat` en `resolver.go`.

### 2. `~` sin expandir en paths operativos

`resolveSingleState` almacenaba paths con `~` (ej: `~/.zshrc`) en `LinkStatus`, y luego `os.Symlink` trataba `~` como literal, creando archivos con nombre `~` en el directorio actual.

**Fix:** Los paths absolutos se almacenan en `LinkStatus` desde el resolver; `~` se usa exclusivamente para display vía `shortDisplayPath()`.

### 3. Nombres de variantes inconsistentes

`DetectVariants()` usaba el source path completo como nombre de variante (`termux/.zshrc`). Para globs como `wilber/*` ya extraía solo `wilber`, pero para archivos concretos usaba todo el path.

**Fix:** `VariantKey()` extrae el prefijo de directorio (`termux/.zshrc` → `termux`), consistente con globs.

### 4. Checker visible en `dots status` y `dots list`

El checker de path.yaml solo se ejecutaba en comandos mutantes (link, unlink, install, adopt). Ahora también se ejecuta en status y list, mostrando warnings/errors para revisión consciente sin bloquear la ejecución.
