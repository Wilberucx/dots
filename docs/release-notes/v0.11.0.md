# Release Note: v0.11.0 — Shell completion, backup refinements, various fixes

**Fecha:** 2026-05-24
**Versión:** `v0.11.0`
**Tipo:** Features / Bug fixes

## 🚀 Nuevas Funcionalidades

### 1. `dots completion {bash|zsh|fish|powershell}`

Nuevo subcomando que genera scripts de autocompletado para los 4 shells principales usando el generador nativo de Cobra.

```
dots completion bash        → source <(dots completion bash)
dots completion zsh         → source <(dots completion zsh)
dots completion fish        → dots completion fish | source
dots completion powershell  → dots completion powershell | Out-String | Invoke-Expression
```

Soporta autocompletado de subcomandos, flags, argumentos y nombres de módulo.

### 2. Mejoras en `dots backup`

- **Pre-flight checks:** Validación de estado del repo antes de ejecutar (rebase/merge/cherry-pick en progreso).
- **Sync mejorado:** Fetch explícito con nombre del remote, mensajes más descriptivos.
- **Flag `--no-verify`:** Salta hooks de git durante el commit.
- **Stage con `git add -A`:** Captura también eliminaciones y archivos nuevos.
- **Push robusto:** Push explícito con remote + HEAD en vez de `git push` genérico.

### 3. Variant swap por módulo

El auto-swap de variantes ahora es granular por módulo en vez de aplicar `--force` globalmente, permitiendo mezclar módulos con distintas variantes activas en un mismo comando.

## 🐛 Bugs Corregidos

### 1. Glob patterns en checker

El checker de existencia de fuentes ahora salta correctamente patterns con `?` y `[` además de `*`.

### 2. Capitalización con caracteres multi-byte

`promptModuleName` ahora usa `unicode.ToUpper()` en vez de aritmética de bytes, evitando corrupción con caracteres no ASCII.

### 3. Marcador `init` con formato YAML válido

El archivo `.dots/config.yaml` ahora usa `version: "1"` en vez de formato TOML inválido (`[dots]\nversion = "1"`).

### 4. Timeout en descargas

`dots install` ahora usa un timeout de 30s en descargas HTTP en vez de quedar bloqueado indefinidamente.

### 5. Directorios ruidosos en `list --backups`

`.cache`, `node_modules`, `.npm`, `.cargo` y `.local/share/Trash` se saltan al buscar archivos `.orig`.
